Professional Self-Assessment

Completing the Computer Science program and developing this ePortfolio has allowed me to transform from a student completing assignments into a computing professional capable of designing, evaluating, and securing real systems. Throughout the program, I developed technical depth in software design, algorithms, and databases while also refining my ability to communicate technical concepts clearly to diverse audiences. The capstone experience required me not only to enhance previous artifacts but also to critically evaluate my own work, identify weaknesses, and implement improvements that align with industry standards. This process strengthened both my technical skill set and my professional identity.

One of the most significant areas of growth during the program has been collaboration and communication. Through code reviews, structured design documents, and narrative reflections, I learned how to explain technical decisions in a way that supports stakeholder understanding and organizational decision making. The informal code review required me to present existing functionality, analyze weaknesses, and justify planned enhancements as if speaking to peers or management. This experience mirrors real-world development environments where engineers must communicate trade-offs, justify architectural decisions, and collaborate effectively within teams.

In the area of software design and engineering, my work on the DriverPass system demonstrates growth in architectural thinking and security awareness. Originally, the system design focused on functionality such as scheduling lessons and tracking student progress. Through enhancement, I strengthened role-based access considerations, clarified system boundaries, and emphasized security responsibilities within the design. This demonstrates my ability to evaluate software architecture not only for functionality but also for maintainability, scalability, and risk mitigation. These are essential competencies in modern IT environments where system reliability and compliance are critical.

My enhancement in algorithms and data structures through the CS300 course planner artifact demonstrates my ability to evaluate trade-offs in data structure selection and runtime performance. By comparing vector-based storage, hash tables, and binary search trees, I gained practical insight into time complexity, efficiency, and scalability. I learned to justify design decisions based on performance characteristics rather than convenience. This directly aligns with the program outcome of designing computing solutions using algorithmic principles while managing trade-offs. The process reinforced the importance of selecting appropriate data structures to ensure optimal performance and long-term maintainability.

In the databases category, the Travlr Getaways full-stack application allowed me to demonstrate database integration, validation, and secure data handling. Through enhancements, I strengthened schema constraints, implemented consistent server-side validation and sanitization, improved error handling, and ensured secure database interactions. These improvements reflect a proactive security mindset that anticipates adversarial exploits and mitigates vulnerabilities at the architectural level. Understanding how improper validation or weak schema enforcement can compromise data integrity has significantly influenced how I approach system design.

Security has become a consistent theme throughout my development as a computer science professional. Rather than viewing security as an afterthought, I now approach system architecture with the assumption that inputs may be malicious and that access must be controlled explicitly.
